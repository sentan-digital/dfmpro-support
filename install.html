まずやること（最短ルート）

  1. 対応バージョンを確認（Creo / OS / DFMPro）
  2. インストーラー（Zip）をダウンロード
  3. 管理者権限でセットアップ実行
  4. Creo起動 → DFMProメニュー確認
サポートに連絡 つまずき集を見る

0. 事前確認

重要（超多いミス）： DFMProは「Creoのメジャーバージョン別（8/9/10/11/12…）」にインストーラーが分かれます。
例：Creo 8 に Creo 9 用を入れても メニューが出ません。まず Creo対応バージョン を確認してください。
管理者権限について： インストール時は 管理者権限 が必要な場合があります。
うまく登録されないときは、まず「セットアップを 右クリック → 管理者として実行」を確認してください。

必要情報（問い合わせ時にも必須）

  • Creoバージョン（例：Creo 8.0 / 9.0 / 10.0 …）
  • OS（Windows 10/11 など）
  • DFMProバージョン（例：v13.x）
  • ライセンス形態（ノードロック / フローティング など）

ヒント：Creoのバージョンは、Creo起動後 ヘルプ → バージョン情報 等で確認できます。

事前に確認しておくこと

  • インストール先PCに対象のCreoが入っているか
  • セキュリティソフト/社内ポリシーでブロックされないか
  • 既存DFMProがある場合、上書き可否（運用方針）
  • Creoが複数入っている場合：どのCreoに導入するか（8/9/10…）

2. インストール

手順（基本）

  1. ダウンロードしたZipを任意のフォルダに保存
  2. Zipを解凍（例：C:\Temp\DFMPro\）
  3. セットアップ（例：Setup.exe）を右クリック → 管理者として実行
  4. ウィザードに従ってインストール
  5. インストール後に対象のCreoを起動
Windowsの警告が出る場合： SmartScreen等でブロックされた場合は「詳細情報」→「実行」を選択してください。
（社内ポリシーで不可の場合は情シスへ）
つまずきポイント（最頻出）：
「インストールは終わったのにCreoにメニューが出ない」場合は、
①管理者として実行したか / ②Creoバージョンの取り違え（8/9/10/11/12） が多いです。
→ つまずき集 を先に確認してください。

Creoが複数入っている場合（メモ）

  • 導入したいCreoのメジャーバージョンに合うインストーラーを選択してください（8/9/10…）。
  • インストール後は、導入対象のCreoを起動して、リボン/メニューの表示を確認します。

（任意）上書き・再インストール時のメモ

  • 既存DFMProがある場合、まずはアンインストール推奨（運用で決める）
  • 社内標準のインストール先がある場合はここに追記

4. 動作確認

  1. Creo起動後、DFMProメニュー / リボンが表示されるか
  2. Rule Manager が起動できるか
  3. 簡単なモデルでチェック実行できるか

もしDFMProが表示されない場合は、まず 「CreoにDFMProが出てこない」 を確認してください。

※スクリーンショット付きにすると強いので、後で差し込みましょう（例：どのリボンに出るか、どのメニュー名か）。

5. よくあるつまずき（仮）

CreoにDFMProが出てこない

  • セットアップを「右クリック → 管理者として実行」したか
  • 対象Creoバージョンの取り違え（8/9/10/11/12など）
  • セキュリティソフト/社内ポリシーでブロックされていないか

よくあるパターン：Creo 8 に Creo 9 用を入れている／Zip解凍が途中で止まっている／管理者権限なしで実行した。

起動時に認証エラー

  • ライセンス形態（ノードロック/フローティング）が合っているか
  • ライセンス期限 / 台数上限
  • フローティングの場合、サーバー疎通（VPN/社内FW/プロキシ）

インストーラーが起動しない

  • Zip解凍が途中で止まっていないか（別フォルダへ解凍し直す）
  • Windowsの保護（SmartScreen）でブロックされていないか
  • ダウンロードが破損していないか（再DL）

6. アンインストール / 再インストール（仮）

※詳細は後でページ化します（例：uninstall.html）。ひとまず“相談が多いポイント”だけ置きます。

  • 再インストール前に、既存DFMProのアンインストールが必要な場合があります（運用で決定）。
  • 同一PCでのアップデート/上書き可否は、配布パッケージの方針に合わせます。
  • 「Creoに出ない」系は、まず 管理者実行 と Creoバージョン を確認してください。
